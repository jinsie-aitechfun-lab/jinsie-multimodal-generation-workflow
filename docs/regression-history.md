# 2026-07 生产稳定性优化

## 1. 本地正常，生产环境异常

### 问题现象

本地开发环境表现正常，但部署 Railway / Docker 后出现：

-   字幕字体不同。
-   字幕换行效果不同。
-   页面首次进入布局异常。
-   部分资源加载体验下降。

### 根因分析

开发环境和生产环境存在差异：

-   操作系统不同。
-   字体环境不同。
-   Docker 镜像环境不同。
-   浏览器缓存策略不同。

### 修复方案

-   Docker 固定中文字体环境。
-   FFmpeg 字幕渲染明确指定字体和样式。
-   增加生产环境验收流程。
-   增加部署后的真实环境验证。

### 防回归措施

生产环境验证不能只依赖本地测试。

# Workflow 生命周期恢复问题

## 2. 旧 Workflow 响应覆盖新 Workflow

### 问题现象

用户创建新 workflow 后：

-   旧请求返回。
-   旧状态覆盖新状态。
-   页面显示错误任务。

### 根因分析

异步请求没有生命周期隔离。

旧请求：

    workflow A
    ↓
    异步恢复
    ↓
    返回

可能覆盖：

    workflow B

的新状态。

### 修复方案

增加：

-   workflowLifecycleGeneration
-   workflowId 校验
-   restore Promise 生命周期隔离
-   final video polling 校验

旧 workflow 完成后不能修改新 workflow 状态。

### 防回归措施

所有异步恢复逻辑必须验证：

    generation
    workflowId
    request marker

# 图片生成相关问题

## 3. 重新生成复用旧图片

### 问题现象

点击"重新生成"：

-   图片没有变化。
-   看起来像重新生成，但实际没有调用模型。

### 根因分析

恢复逻辑和主动生成逻辑边界混淆。

旧逻辑：

    发现已有文件
    ↓
    认为任务成功
    ↓
    直接恢复

导致 refresh 被当成 restore。

### 修复方案

区分：

恢复：

    已有任务
    ↓
    继续恢复状态

重新生成：

    创建新 task
    ↓
    调用 provider
    ↓
    生成新候选

### 防回归措施

refresh 流程必须验证：

-   新 task 创建。
-   provider 被调用。
-   候选内容发生变化。

## 4. Selector 固定选择第一张图片

### 问题现象

图片审核阶段：

-   总是选择 candidate A。
-   高评分候选无法成为最终选择。

### 根因分析

reconcile 阶段覆盖 selector 结果：

    selector result

    ↓

    被旧 selection 覆盖

    ↓

    fallback refs[0]

导致第一张图片成为默认选择。

### 修复方案

保留 selector authoritative state：

-   selected_asset_ref
-   candidate scores
-   selection reason

不再使用固定第一候选。

### 防回归措施

增加 selector 回归测试：

-   candidate B 得分更高。
-   最终必须选择 B。

# 历史视频问题

## 5. 历史视频封面重新进入丢失

### 问题现象

第一次生成：

-   封面正常。

重新进入：

-   出现占位符。

等待一段时间：

-   又恢复。

### 根因分析

异步 metadata 合并顺序错误。

旧 fallback：

    poster fallback

覆盖：

    真实 thumbnail

导致 localStorage 保存错误数据。

### 修复方案

调整 metadata 合并优先级：

    真实 posterUrl
    >
    thumbnail
    >
    fallback

### 防回归措施

验证：

-   首次进入。
-   刷新页面。
-   重新打开 Studio。

三种情况封面一致。

# 字幕与配音问题

## 6. 字幕和配音部分错位

### 问题现象

部分长文本：

-   字幕提前。
-   字幕滞后。
-   句子和声音无法完全对应。

### 根因分析

原流程：

    一个 TTS segment

    ↓

    生成一个长音频

    ↓

    字幕拆成多个 cue

    ↓

    平均分配时间

但是：

真实朗读：

-   标点停顿不同。
-   语速不同。
-   长短句耗时不同。

平均切分无法对应真实声音。

### 修复方案

调整流程：

    文本拆分

    ↓

    每段单独 TTS

    ↓

    ffprobe 获取真实 duration

    ↓

    一个 audio segment

    ↓

    一个 subtitle cue

字幕时间完全来自真实音频。

### 防回归措施

禁止：

    一个音频
    → 多字幕 cue
    → 平均分配时间

# 移动端与加载体验

## 7. Landing 进入 Studio 白屏

### 问题现象

点击进入 Studio：

-   没有反馈。
-   等待动态资源加载。
-   用户认为页面卡死。

### 根因分析

Studio 使用懒加载：

    router.push

    ↓

    加载 chunk

    ↓

    渲染页面

期间没有 loading 状态。

### 修复方案

Hero 主按钮增加：

-   navigating 状态。
-   loading 动画。
-   disabled。
-   loading 文案。

------------------------------------------------------------------------

## 8. Studio 首次进入横向撑宽

### 问题现象

首次进入：

-   页面可能横向滚动。
-   布局抖动。

### 根因分析

flex/grid 默认最小宽度：

    min-width:auto

导致内容撑开容器。

### 修复方案

增加：

-   min-width:0
-   max-width:100%
-   minmax(0,1fr)

保证安全收缩。

# 核心原则

AI Video Studio 已从 Demo 阶段进入产品稳定阶段。

后续优化重点：

不是增加更多功能，而是保证：

-   状态可靠。
-   环境一致。
-   异步安全。
-   用户体验稳定。
-   修改不会破坏已有能力。
